<objective>
Fix the CSRF validation bug in the contact form, add error handling for SMTP failures, and add gunicorn to src/requirements.txt.

Purpose: The contact form CSRF token is generated but never validated because HTML input `name` attributes use capitalized names (`Name`, `Email`, `Message`) while WTForms expects lowercase. This means `form.validate_on_submit()` always returns False and any POST succeeds without CSRF check. Additionally, `mail.send()` has no error handling — any SMTP failure produces an unhandled exception and leaks a stack trace. Gunicorn is also missing from src/requirements.txt but required by the Dockerfile.

Output:
- src/templates/contact.html — input name attributes corrected to lowercase, form action fixed
- src/main.py — mail.send() wrapped in try/except with abort(500), `abort` import added
- src/requirements.txt — gunicorn added
</objective>

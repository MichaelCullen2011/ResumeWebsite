|----------|-------------|
| Host filesystem → Docker container | Environment variables injected via .env file or docker run -e flags |
| .env.example in git | Placeholder file is public; must contain no real secrets |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-06 | Elevation of Privilege | Container running non-existent module (app:app) silently failing | mitigate | Task 1 corrects CMD to src.main:app; container now starts correctly and process supervision works as expected |
| T-02-07 | Denial of Service | Unused Redis container consuming resources in local dev | accept | Task 2 removes Redis entirely; low severity since this only affects local dev environment |
| T-02-08 | Information Disclosure | CLAUDE.md documenting env var names (not values) | accept | Documenting variable names is standard practice; actual values are in .env (gitignored); no risk from name disclosure |
</threat_model>
